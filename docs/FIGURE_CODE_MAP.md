# Figure-to-code map

The map below describes the scope of the supplied notebooks. Exact panel
assembly in the manuscript also includes manual schematics and layout work.

| Notebook | Figure scope represented in the active code |
| --- | --- |
| `01_behavior.ipynb` | Fig. 1C-D, Fig. 3A-B, Fig. S1A-B |
| `02_single_neuron.ipynb` | Fig. 1E-F and Fig. S2-S3 single-neuron summaries/examples |
| `03_statespace.ipynb` | Fig. 2 state-space/temporal-generalization panels and Fig. S4-S5 state-space panels |
| `04_population.ipynb` | Fig. 2 population panels, Fig. 3F-I, Fig. 4, and corresponding Fig. S4-S6 panels |
| `05_colorshape.ipynb` | Fig. 5 behavior/population panels and Fig. S7 analyses |
| `06_model_prediction.ipynb` | Derived data analyses for Fig. 6G-H only |

Not computationally generated by this package:

- task schematics and conceptual diagrams;
- imaging field-of-view, electrode-layout and waveform montage source panels;
- RNN architecture/training/checkpoints and Fig. 6A-F producers;
- raw neural-data preprocessing and the upstream producers of the supplied
  processed pickle objects.

The reproducible artifact for plots whose `savefig` line remains commented is
the executed notebook with its inline output under
`results/executed_notebooks/`. Active save calls write to `results/figures/`.
